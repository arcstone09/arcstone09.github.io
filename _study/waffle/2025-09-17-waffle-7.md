# Account

- **waffle GitHub organization** : https://github.com/orgs/wafflestudio/people

  - **Organization** : 여러 명이 협업하는 **팀·회사·동아리 전용 GitHub 공간**
  - 멤버가 되면 해당 Organization 안의 레포지토리에 접근할 수 있고 부여된 권한에 따라 읽고 쓰기 가능

- **waffle notion** : https://www.notion.so/1be70e37e9c781d6ba2ed55ad2ca19c3

  - **waffle Infra notion** : https://www.notion.so/Infra-1be70e37e9c7819d940ef5e28d0d57fa

  - notion 기본 층위구조

  1. **계정(Account)**
     - 이메일 1개당 1개 계정
     - 계정 안에 여러 **워크스페이스**를 가질 수 있음
  2. **워크스페이스(Workspace)**
     - 팀/조직 단위 협업 공간
     - 여기서 초대받으면 → 워크스페이스 전체에 접근 가능 (단, 페이지별 세부 권한은 따로 있을 수 있음)
  3. **페이지(Page)**
     - 워크스페이스 안에 있는 문서 단위
     - 워크스페이스 멤버가 아니어도 **페이지 단위로만 초대** 가능 (게스트 초대 기능)
  4. **하위 페이지(Sub-page)**
     - 특정 페이지 안에 또 다른 페이지
     - 상위 페이지 권한을 그대로 상속하거나, 따로 공유 설정 가능

- **waffle AWS IAM account** : (팀에 소속되고 나면 IAM 계정 발급될듯)

  - alias : wafflestudio / username : arcstone09 / pw : h~*

  - **IAM (Identity and Access Management)** = AWS에서 **사용자와 권한을 관리하는 서비스**

    **IAM 계정** : 루트 계정(=AWS 가입한 메인 계정)에서 만들어서 팀원·개발자에게 주는 **개별 사용자 계정**

  - Root 계정은 AWS 가입 시 처음만든 (메일주소, 비밀번호) 로 로그인한다.
  
    - 고유한 Account ID 12 자리가 부여된다. 
    - **Account Alias (계정 별칭)** 이란, 이 Account ID 대신 쓸 수 있는 이름이다. 즉, 메일=Root 계정 당 1개씩 존재한다.
  
  - IAM 계정은 (접속하려는 Root 계정의 ID or 별칭, IAM 사용자 이름, 비밀번호) 로 접속한다.
