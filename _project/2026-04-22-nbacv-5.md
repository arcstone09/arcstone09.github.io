# Vast AI

Vast.ai는 aws보다 저렴하게 gpu 및 스토리지를 대여할 수 있는 도커 기반의 클라우드 서비스이다. 사용 방법은 아래와 같다.

## Renting GPU Instance

1. GPU 종류 선택

- RTX 3090, 4090, 5090, H100, 200 등 사용할 GPU의 종류를 선택한다. GPU 종류에 따라 다음이 다르다.
- VRAM : GPU의 RAM. 우리가 사용할 RTX 3090의 경우 16GB
- TFLOPS : 초당 연산 속도. Trillion Floating Point Operations per Second의 약자로 초당 몇 번 연산하는지를 조 단위로 나타낸다. 우리가 사용할 RTX 3090의 경우, 35.3 TFLOPS.

![image-20260422202953779](../../images/2026-04-22-nbacv-5/image-20260422202953779.png)

2. 여러 개의 GPU 중 하나를 선택한다.

- 같은 RTX 3090 GPU 라도 네트워크 속도, Disk (로컬 저장공간), CPU/RAM, Reliability (서버 안정성) 등이 다르다.
- 좋은 머신의 경우, DLPerf 높음, Network 빠름, Reliability 99% 이상, Price 낮음 등을 기준으로 고르면 된다.

![image-20260422202759902](../../images/2026-04-22-nbacv-5/image-20260422202759902.png)

3. Template을 선택한다.

- 도커 이미지를 선택한다. 

- PyTorch(Vast) 이미지는 Ubuntu + CUDA + PyTorch + 기본 라이브러리를 포함하고 있다.

- CUDA는 CPU는 GPU를 직접 쓸 수 없기 때문에 중간에서 연결하는 언어이다. 

- ```scss
  [하드웨어]
  GPU (RTX 3090)
  
  [드라이버]
  NVIDIA Driver
  
  [CUDA]
  GPU 연산 인터페이스
  
  [프레임워크]
  PyTorch / TensorFlow
  
  [환경]
  Docker (컨테이너)
  
  [템플릿]
  PyTorch (Vast) ← 지금 고르는 것
  ```

![image-20260422202709698](../../images/2026-04-22-nbacv-5/image-20260422202709698.png)

4. Storage (Container + Volume) 를 선택한다.

- Container Disk : 우리가 대여한 RTX 3090의 디스크를 실제로 여러 사용자들이 나누어 사용한다. 이 중 우리에게 할당된 값을 설정하는 것이다. 기본 16GB로 설정되어 있다. 

- Container Disk는 대용량 영상을 저장하기에 작고 인스턴스 종료시 삭제되므로 추가적인 볼륨이 필요하다.
- Volume : 영구 저장 가능하고, GPU 인스턴스를 바꿔도 새로운 인스턴스에 붙여 사용할 수 있다. 100GB로 설정하고 이미 존재하는 volume이 없기 때문에 Create local volume을 선택한다.

![image-20260422202624531](../../images/2026-04-22-nbacv-5/image-20260422202624531.png)

5. 결제

- 4번까지 설정 후 선택했던 GPU의 Rent 버튼을 다시 누르면 결제를 한다. 이 때 입력하는 값은 선 결제 값으로 해당 액수의 금액이 미리 충전되고 GPU를 사용할수록 차감된다.

![image-20260422200937945](../../images/2026-04-22-nbacv-5/image-20260422200937945.png)

## Setup Jupyter Direct HTTPS

Renting을 마치고 나면 다음과 같은 안내가 나온다.

![image-20260422202200437](../../images/2026-04-22-nbacv-5/image-20260422202200437.png)

- 우리가 대여한 GPU 인스턴스를 실행하면, 인스턴스 안에서 도커 컨테이너 안에서 `jupyter notebook` 이라는 프로그램이 실행된다.
- Jupyter는 브라우저에서 파이썬 코드 실행하는 웹 프로그램이다. Jupyter 프로그램이 하는 일은 크게 두 가지이다.
  - 웹 서버 역할 : 브라우저의 요청을 받고, 코드/결과를 HTML로 보여준다.
  - 코드 실행 컨트롤러 : Python 커널을 실행하고 결과를 반환한다. 셀 단위로 코드 실행이 가능하다.

위 안내는 우리의 브라우저가 vast.ai 서버(Jupyter)를 가짜일 수도 있다고 의심해서 막고 있는 상황에 대한 설명이다. 우리가 브라우저를 통해 웹 서버에 접속하려고 하면, 브라우저는 웹 서버에게 인증서를 제출하도록 하고, 브라우저가 이 인증서를 확인하고 OK이면 접속을 하게 된다. 기본적으로 공인된 인증서(CA)만 믿는데, jupyter notebook에서는 Jupyter 서버를 띄우고 HTTPS 쓰려고 자체 인증서(self-signed) 생성하는데, 이 인증서는 공인 기관이 발급한 게 아니라서 브라우저가 신뢰하지 않는다. 그래서 저 안내가 하는 말은 이 인증서를 브라우저에 수동으로 직접 등록해서 '이건 믿어도 된다'고 우리가 알려주라는 이야기이다.

실제로 jupyter terminal에 접속하려고 하면 다음과 같은 안내가 나온다.

![image-20260422210744124](../../images/2026-04-22-nbacv-5/image-20260422210744124.png)

- 웹 서버란, 브라우저에서 http request를 받고, 무언가를 수행하여 그 결과를 다시 브라우저에 http response로 주는 프로그램이다. jupyter notebook도 웹 서버인데 그 이유는 `print(1+1)` 이라는 코드를 입력하고 브라우저에서 실행 버튼을 누르면 요청이 서버로 감 → 서버가 Python 코드 실행 → 결과 `2` 생성 → 결과를 브라우저로 다시 보냄 → 화면에 출력 과정이 진행되기 때문이다. 

- 마찬가지로, 인스턴스의 터미널도 브라우저를 통해 접근 가능하다. jupyter terminal을 브라우저에서 실행하면 다음의 일들이 일어난다. 

- ```scss
  [브라우저]
     ↓ "ls 입력"
  [Jupyter 서버 (웹서버)]
     ↓
  [bash 쉘 실행]
     ↓ 결과 생성
  [Jupyter 서버]
     ↓
  [브라우저에 출력]
  ```

## Data Transfer

- jupyter terminal 에서 git clone 하여 github repo에서 서버로 코드를 내려 받는다. 
- 로컬에서 scp 명령어를 이용해 서버로 영상을 옮긴다.

📝 **SSH Key**

SSH는 네트워크로 다른 컴퓨터에 안전하게 접속해서 명령을 실행하는 프로토콜이다. 모든 통신이 암호화되어 전달된다. 이 프로토콜에 맞게 구현한 프로그램이 ssh, OpenSSH 등이다. 

이런 ssh 프로그램은 다음의 일을 수행하도록 코딩되어 있다.

- 원격 접속 

  - ```bash
    ssh root@server_ip -p 12345
    ```

- 명령 실행 

  - ```bash
    ls
    python train.py
    ```

  - 내 컴퓨터에서 서버를 조종

- 파일 전송

  - ```bash
    scp file.txt server:/path/
    ```

  - 로컬 ←→ 서버 간 파일 이동

- 이를 위해, TCP 연결, 암호화 채널 생성, 인증 등의 과정을 하도록 코딩되어 있다. 

이렇게 ssh 프로그램을 사용하여 SSH 통신을 하기 위해서는 SSH Key가 필요하다. 

SSH Key는 (private key, public key) 쌍으로 이루어져 있다. 로컬에서 `ssh-keygen -t ed25519` 를 통해 생성할 수 있다. ed25519는 암호 알고리즘 이름이다. 생성된 private key는 ~/.ssh/id_ed25519에 저장되어 있다. private key는 절대로 외부에 유출되어서는 안된다. public key는 ~/.ssh/id_ed25519.pub 에 저장되어 있으며 이 값을 서버에 등록한다. 즉, public key는 접속하고자 하는 서버에 등록된 내 정보이다. 

ssh로 서버에 접속을 시도하면, 서버는 진짜 내가 맞는지를 묻고, 우리는 private key를 바탕으로 암호를 전달하고, 서버는 public key로 이를 검증한다. 서버는 private key를 절대 모르지만, public key를 통해서 검증은 가능하다. 이를 비대칭 암호 (public-key cryptography) 라 한다.



Vast.ai의 인스턴스 대시보드에서 ssh key를 등록한다. 그럼 ssh 연결이 가능한 포트 주소가 다음처럼 나온다.

![image-20260422220951331](../../images/2026-04-22-nbacv-5/image-20260422220951331.png)

이제 scp 명령어를 이용해 영상을 서버에 올릴 수 있다.

```bash
scp -P 33457 -r ~/Desktop/snupi/nba_cv/data root@136.59.129.136:/workspace/ 
```

이처럼 바로 올리면 시간이 많이 소요되므로 압축 후 올린다.

```bash
# 로컬
tar -czf data.tar.gz data  
scp -P 33457 -r ~/Desktop/snupi/nba_cv/data.tar.gz root@136.59.129.136:/workspace/ 

# vast 서버
tar -xzf data.tar.gz
```

그런데 이 방법도 시간이 오래 걸려 로컬에서 필요한 라이브러리들에 대한 requirement.txt를 만들고 서버에서 git pull하여 다운 받은 다음, 영상들은 다시 직접 다운 받는 방법을 택했다. 

```bash
# vast 서버
pip install -r requirements.txt 
apt update
apt install -y tesseract-ocr # pytesseract는 Python 라이브러리 + 외부 프로그램 2개가 같이 있어야 동작
```

## Workflow

- 로컬에서 코드 수정 → git push → pr → git merge (main 브랜치로) → 로컬에서 git pull 하여 main 병합 → 서버에서 git pull (처음에만 clone) → 서버에서 모델 학습 (이때 로컬에서 ssh로 접속하여 작업)  → 서버에 파라미터 파일로 저장됨 → scp로 로컬에 내려 받음
