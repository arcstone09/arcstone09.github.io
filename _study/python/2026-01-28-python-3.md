# Anaconda

Anaconda란 파이썬 + 라이브러리 묶음을 통째로 분리한 독립 공간이다. Anaconda를 설치하여 이용하면 다음과 같이 독립적인 프로젝트 구조를 갖출 수 있게 된다.

```csharp
Anaconda 설치
│
├─ base  (기본 환경, 웬만하면 안 건드림)
│  ├─ python
│  └─ pip 
│
├─ pose-env
│  	├─ pip3
│   ├─ python
│   ├─ numpy
│   ├─ mediapipe
│   └─ opencv
│
└─ cs231n-env
  	├─ pip3
    ├─ python
    ├─ torch
    ├─ torchvision
    └─ jupyter
  
Homebrew Python
 ├─ python3
 └─ pip3  ← 또 다른 pip

macOS system Python
 ├─ python3
 └─ pip3  ← 또 다른 pip  
 
```

## command

```shell
# 환경 만들기
conda create -n cs231n python=3.10
# 환경 활성화
conda activate cs231n
# 패키지 설치
python -m pip install mediapipe opencv-python
# 패키지 삭제
pip uninstall mediapipe ultralytics opencv-python
# 코드 작성 후 실행
python your_pose_code.py
# 환경 나가기
conda deactivate
# 현재 실행될 python 인터프리터 경로
which python
```

일반적으로 anaconda를 설치하면 로컬 파이썬이 가려진다. 로컬 파이썬을 쓰고 싶다면 venv를 사용하거나 venv를 사용하고 싶지 않다면 가려진 .zshrc를 수정해야 한다.



# venv

venv는 파이썬이 기본으로 제공하는 가상환경” 기능이다. 파이썬 + pip + 라이브러리를 한 폴더에 격리하는 것이다. venv는 새 python을 설치하지 않는다. 기존 로컬 python을 **그대로 복사/참조**, 단지 **라이브러리 경로만 분리**한다. 그래서 빠르고 가볍다.

```csharp
pose-env/
├─ bin/
│   ├─ python   ← 이 환경 전용 python
│   └─ pip
├─ lib/
│   └─ python3.x/
│       └─ site-packages/
└─ pyvenv.cfg
```

## command

```shell
# 가상 환경 만들기
python3 -m venv cs231n-env
# 가상 환경 활성화
source pose-env/bin/activate
# 패키지 설치
python -m pip install numpy
```

source 하는 순간 이 순간:

- `PATH`가 바뀜
- `python` → `pose-env/bin/python`
- `pip` → `pose-env/bin/pip`
